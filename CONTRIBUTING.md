# Contributing

Issues and pull requests welcome. The Knack Starter Pack is small on purpose; contributions that fit that scope land fastest.

## Issues

Three kinds we welcome:

- **Bug reports.** Tell us the OS + shell, what you ran, what happened, what you expected. Paste the failing output if you can.
- **Worked-example contributions.** See the open issue inviting real-world `idea-clarifier` runs. Anonymise freely.
- **Improvements to install notes.** Especially for Windows, where edge cases (OneDrive Known Folder redirection, execution policy, non-interactive flows) keep surfacing.

What we'd usually decline, with thanks:

- New skills inside this pack. Two skills is the deliberate scope. New scope ideas are better as a new pack proposal — email `hello@knack.run`.
- Bundling MCP servers, agents, or slash commands. By design, the starter pack stays minimal.

## Pull requests

- Small, focused PRs. Big rewrites: open an issue first.
- The pack source is mirrored from the Knack monorepo; changes accepted here are propagated back. You don't need to think about that, we will.
- British English in user-facing copy. Plain words, no marketing nouns ("leverage", "robust", "synergy", "unlock").
- No em-dashes (—) in public copy. Use commas, full stops, parentheses, or en-dashes (–) for ranges.
- One commit per logical change is fine; many tiny commits also fine. We squash on merge if it helps.

## Larger contributions and partnerships

For new-pack proposals, partnership questions, affiliate enquiries, or anything that doesn't fit an issue:
