# INPUT: first-skill-template walkthrough

> **Composited example.** Anonymised from the kind of "how do I make my first Claude Code skill" request that comes up in indie-hacker Discords.

## What the user typed to start the skill

> *"I want to make a skill for my weekly build-in-public posts. Every Friday I write a post on X about what shipped that week, what didn't, and what I learned, but I always stare at a blank page for half an hour first. I want to scaffold a skill that turns my week's notes into a draft post. Help me set up the file."*

## What the user has, before this skill runs

- **Idea for the skill:** turn a week's worth of notes into a build-in-public Friday post.
- **Where they want it:** project-local, in their indie-hacker repo (they're at the project root).
- **Initial name they propose (in their head, not yet typed):** "weeklyReview" (the skill will reject this on the kebab-case rule).
- **Initial description (in their head):** "A skill for productivity" (the skill will reject this as a category, not a trigger).

The skill's first move is to push back on both, demand specifics, and only then scaffold.
